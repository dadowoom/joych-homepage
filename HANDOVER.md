## 2. 완료된 주요 기능

### 2-1. 홈페이지

- 히어로 영상 슬라이드 (mp4 자동 재생, 영상 끝나면 다음 슬라이드로 전환)
- 퀵메뉴 (DB 관리, 관리자가 직접 편집 가능)
- 조이풀TV 섹션 (최신 예배영상 자동 표시)
- 교회소식/공지사항 (DB 연동)
- 비전 섹션, 갤러리, 관련기관 (DB 연동)
- 반응형 디자인 (모바일/태블릿/PC)

### 2-2. 동적 메뉴 시스템

- 1단/2단/3단 메뉴 구조를 관리자가 직접 편집
- 각 메뉴 페이지 타입: `image` (이미지), `gallery` (갤러리), `board` (게시판), `youtube` (영상 목록), `editor` (블록 에디터)
- 드래그 앤 드롭 순서 변경

### 2-3. 블록 에디터

- 텍스트 블록 (H1/H2/H3/본문), 이미지 블록 (1장/2장/3장), 유튜브 블록, 버튼 블록, 구분선 블록
- 관리자 모드에서 블록 추가/수정/삭제/순서 변경
- 블록 저장 후 즉시 반영 (invalidate 처리 완료)

### 2-4. 시설 예약

- 시설 목록/상세 페이지 (DB 연동)
- 달력 기반 예약 가능 날짜 확인
- 시간 슬롯 선택 (운영시간 기반 자동 생성)
- 예약 신청 폼 (성도 로그인 필요)
- 내 예약 현황 페이지
- 관리자 예약 승인/거절

### 2-5. 성도 회원 시스템

- 이메일/비밀번호 회원가입 (관리자 승인 후 활성화)
- 로그인/로그아웃 (세션 쿠키 방식, JWT 서명)
- 로그인 성공 시 상단 바 이름 즉시 표시 (invalidate 처리 완료)
- 마이페이지 (개인정보 수정)
- 관리자 교적부 관리 (성도 목록, 승인/거절, 직분/부서 등록)

### 2-6. 보안
- 관리자 자격증명 환경변수 이전 완료 (ADMIN_USERNAME, ADMIN_PASSWORD, ADMIN_OPEN_ID)
- 파일 업로드 MIME 타입 화이트리스트 검증 (server/routers/cms/upload.ts)
- 이미지 최대 10MB, 영상 최대 100MB 제한
- 성도 비밀번호 bcrypt 해시 저장
- 보안 헤더 (Helmet.js)

## 5. 보안 주의사항

### 관리자 자격증명
관리자 자격증명은 환경변수로 완전히 이전되었습니다. 코드에 하드코딩된 자격증명은 없습니다.

```
ADMIN_USERNAME  — 관리자 로그인 아이디
ADMIN_PASSWORD  — 관리자 로그인 비밀번호
ADMIN_OPEN_ID   — Manus OAuth 연동용 관리자 식별자
```

환경변수는 Manus 관리 UI → Settings → Secrets 탭에서 관리합니다.

### 인증 시스템 구조
관리자 인증과 성도 인증은 완전히 별개의 시스템입니다.

| 구분 | 쿠키 이름 | 서명 방식 | 구현 위치 | 비고 |
|---|---|---|---|---|
| Manus OAuth 관리자 로그인 | `app_session_id` | Manus 플랫폼 서명 (`sdk.signSession`) | `server/_core/oauth.ts` | Manus 계정으로 로그인 시 발급 |
| 자체 ID/PW 관리자 로그인 | `app_session_id` | Manus 플랫폼 서명 (`sdk.signSession`) | `server/routers/auth.ts` → `adminLogin` | ENV의 ADMIN_USERNAME/ADMIN_PASSWORD 검증 후 발급. `users` 테이블 admin 계정과 연결 |
| 성도 로그인 | `church_member_session` | JWT (`JWT_SECRET` 환경변수) | `server/routers/members.ts` | `church_members` 테이블 기반 |

> **주의:** 쿠키 이름 `app_session_id`는 `shared/const.ts`의 `COOKIE_NAME` 상수로 관리됩니다. 두 관리자 로그인 방식 모두 동일한 쿠키를 사용하므로, 어느 방식으로 로그인하든 관리자 권한이 부여됩니다.

### 파일 업로드 보안
서버에서 MIME 타입 화이트리스트 검증을 수행합니다 (`server/routers/cms/upload.ts`). 허용되지 않은 파일 형식은 업로드가 거부됩니다.

### 관리자 페이지 URL 보안

관리자 페이지 URL은 `/admin_joych_2026`입니다. `/admin`은 의도적으로 404 처리됩니다. 이 URL을 외부에 공개하지 마세요. 필요 시 URL을 변경하려면 `client/src/App.tsx`에서 수정합니다.

### 성도 비밀번호 저장
성도 비밀번호는 bcrypt로 해시하여 저장됩니다. 평문 비밀번호는 DB에 저장되지 않습니다.

## 6. 운영 주의사항

### 데이터베이스

- **DB는 복구 불가능합니다.** `DELETE`, `DROP`, `TRUNCATE` 명령 실행 전 반드시 백업을 진행하세요.
- 스키마 변경 시 반드시 `pnpm db:push`를 실행해야 합니다. 코드만 수정하면 실제 DB에 반영되지 않습니다.
- 성도 데이터는 소프트 삭제(soft delete) 방식으로 관리됩니다. `status` 컬럼으로 상태를 관리하며, 실제 삭제는 하지 않습니다.

### 파일 업로드

- 모든 이미지/파일은 S3(Manus CDN)에 저장됩니다. DB에는 URL만 저장합니다.
- `client/public/` 폴더에 이미지를 넣으면 배포 시 타임아웃이 발생할 수 있습니다. 반드시 S3를 사용하세요.

### 환경변수

- 환경변수는 절대 코드에 직접 입력하지 않습니다.
- 서버 코드에서는 `server/_core/env.ts`를 통해서만 접근합니다.
- `VITE_` 접두사가 붙은 변수만 프론트엔드에서 접근 가능합니다.

### `server/_core/` 폴더
이 폴더는 Manus 플랫폼 프레임워크 코어입니다. **꼭 필요한 경우가 아니면 수정하지 마세요.** 수정 시 인증, 세션, API 전체가 동작하지 않을 수 있습니다.

## 7. 코드베이스 구조 요약

새로운 기능을 추가할 때 참고해야 할 핵심 파일들입니다.

| 작업 | 수정해야 할 파일 |
|---|---|
| DB 테이블 추가/변경 | `drizzle/schema.ts` → `pnpm db:push` |
| DB 쿼리 함수 추가 | `server/db/{기능명}.ts` |
| API 엔드포인트 추가 | `server/routers/{기능명}.ts` |
| 새 페이지 추가 | `client/src/pages/NewPage.tsx` → `client/src/App.tsx`에 라우트 등록 |
| 재사용 컴포넌트 추가 | `client/src/components/NewComponent.tsx` |
| 전역 스타일 변경 | `client/src/index.css` |
