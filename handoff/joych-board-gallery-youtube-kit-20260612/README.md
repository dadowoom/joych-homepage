# 기쁨의교회 게시판/갤러리/유튜브 이식 키트

이 폴더는 다른 교회 홈페이지 프로젝트에 기쁨의교회에서 만든 핵심 기능 3가지를 이식하기 위한 참고 패키지입니다.

포함 범위:
- 게시판 관련 기능: 공지형 게시판, 자유게시판, 주보 보기, 주보 광고신청, 자막 신청, 탐방 신청, 기도 요청, 관리자 접수 관리
- 갤러리 관련 기능: 최근 행사 사진, 앨범 단위 사진 업로드, 사진 여러 장 묶음, 앨범 순서 변경, 사진 순서 변경, 라이트박스
- 유튜브 링크 기능: 메뉴 타입 `youtube`, 유튜브 플레이리스트, 영상 URL/ID 등록, 공개 영상 목록/상세 표시

사용 방식:
1. `CODEX_PROMPT.md`를 다른 Codex 작업자에게 그대로 붙여넣습니다.
2. `source-files/` 폴더를 참고 자료로 제공합니다.
3. 대상 프로젝트의 라우팅, 인증, DB, 업로드 저장소 구조를 먼저 확인합니다.
4. 이 키트의 파일을 그대로 덮어쓰지 말고, 기능 단위로 필요한 로직을 이식합니다.
5. DB 테이블은 대상 프로젝트의 스키마에 맞춰 마이그레이션을 새로 설계합니다.

중요 주의:
- 이 패키지에는 `.env`, 서버 비밀번호, DB 비밀번호, OAuth 키, S3 키 같은 민감정보가 없습니다.
- 다른 프로젝트에 적용할 때도 민감정보를 코드, README, 커밋, 이식 문서에 넣지 마세요.
- 현재 기쁨의교회 코드는 React, tRPC, Drizzle, MySQL, S3 호환 업로드 구조를 기준으로 합니다.
- 대상 프로젝트가 Next.js, REST API, Prisma, Supabase 등 다른 구조라면 UI/흐름/DB 계약만 참고하고 구현은 대상 구조에 맞춰 다시 작성하세요.

핵심 의존성:
- 프론트엔드: React, Wouter, TanStack Query, tRPC React, Lucide React, @dnd-kit/core, @dnd-kit/sortable
- 서버: Express, tRPC Server, Zod, Drizzle ORM, MySQL2
- 업로드: 현재 프로젝트는 S3 호환 저장소를 사용합니다. 대상 프로젝트가 로컬 업로드면 `server/storage.ts`와 `server/routers/cms/upload.ts`를 해당 방식으로 바꾸세요.

추천 이식 순서:
1. 공통 기반: 라우터 연결, 권한 절차, 업로드 저장소, `contentValidation`
2. DB 스키마: 게시판/갤러리/유튜브 테이블
3. 게시판: 공지/자유게시판/접수형 게시판/주보
4. 갤러리: 공개 목록, 앨범 상세, 관리자 업로드, 정렬
5. 유튜브: 플레이리스트/영상 DB, 관리자 등록, 공개 렌더링
6. 브라우저 검증: 데스크톱/모바일, 로그인/비로그인, 관리자/일반 권한
