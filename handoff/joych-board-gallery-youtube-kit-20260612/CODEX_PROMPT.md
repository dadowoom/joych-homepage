# 다른 Codex 작업자에게 붙여넣을 지시문

이 작업은 다른 교회 홈페이지 프로젝트에 기쁨의교회 홈페이지에서 만든 3가지 기능을 이식하는 작업입니다.

이식 대상 기능:
1. 게시판 관련 기능
   - 공지형 게시판
   - 성도 자유게시판
   - 주보 보기
   - 주보 광고신청
   - 자막 신청
   - 탐방 신청
   - 기도 요청
   - 관리자 접수 관리
2. 갤러리 관련 기능
   - 최근 행사 사진
   - 앨범 제목 입력
   - 사진 여러 장을 한 앨범으로 업로드
   - 앨범 목록
   - 앨범 상세
   - 사진 크게 보기
   - 관리자/권한자 전용 업로드
   - 앨범 순서 변경
   - 사진 순서 변경
3. 유튜브 링크/영상 기능
   - 메뉴 타입 `youtube`
   - 플레이리스트 생성/연결
   - 유튜브 URL 또는 영상 ID 등록
   - 영상 목록/상세 표시
   - 관리자 영상 순서 변경

절대 주의:
- 대상 프로젝트의 기존 디자인/UI/레이아웃/색상/글자 배치를 불필요하게 바꾸지 마세요.
- 기존 기능을 삭제하거나 축소하지 마세요.
- 사용자 또는 다른 작업자가 만든 변경사항을 되돌리지 마세요.
- `.env`, 서버 비밀번호, DB 비밀번호, OAuth 키, API 키, S3 키를 코드/문서/Git에 남기지 마세요.
- 이 키트의 파일을 그대로 덮어쓰지 말고, 대상 프로젝트 구조에 맞춰 필요한 로직만 이식하세요.

먼저 할 일:
1. 대상 프로젝트에서 `git status --short --branch`를 확인하세요.
2. 대상 프로젝트의 라우팅, 관리자 페이지, 인증/권한, DB ORM, 업로드 저장소를 확인하세요.
3. 기쁨의교회 키트의 `FILE_MAP.md`를 읽고 필요한 파일만 좁게 비교하세요.
4. 작업 범위를 게시판, 갤러리, 유튜브 중 하나씩 나눠 진행하세요.

추천 작업 순서:
1. 공통 기반
   - 권한 절차
   - 업로드 API
   - 파일 URL 안전성 검사
   - 메뉴 타입 `board`, `gallery`, `youtube`
2. 게시판
   - 공개 목록/상세
   - 작성/수정/삭제
   - 관리자 상태 관리
   - 주보 여러 이미지
   - 접수형 게시판
3. 갤러리
   - 앨범 목록
   - 앨범 상세
   - 사진 업로드
   - 앨범/사진 순서 변경
4. 유튜브
   - 플레이리스트
   - 영상 URL/ID 저장
   - 공개 렌더링
   - 관리자 정렬

- 주보 광고신청, 자막 신청, 탐방 신청 접수와 관리자 확인

- 앨범 순서와 사진 순서 변경
