# 다른 Codex 작업용 지시문

이 작업은 다른 교회 홈페이지 프로젝트에 기쁨의교회 홈페이지의 CMS, 메뉴 편집, 게시판형 UI/UX, 관리자 기능 패턴을 최대한 비슷하게 이식하는 작업입니다.

참고 패키지:
- `source-files/` 폴더에 기쁨의교회 홈페이지에서 사용 중인 핵심 파일들이 들어 있습니다.
- 이 파일들은 “참고 및 부분 이식용”입니다. 대상 프로젝트에 무작정 덮어쓰지 마세요.

절대 주의:
- 기존 프로젝트의 디자인/UI/레이아웃/색상/글자 배치를 불필요하게 바꾸지 마세요.
- 기존 기능을 삭제하거나 축소하지 마세요.
- 사용자 또는 다른 작업자가 만든 변경사항을 되돌리지 마세요.
- `.env`, 서버 비밀번호, OAuth 키, API 키, DB 비밀번호를 코드/문서/Git에 남기지 마세요.
- DB 변경이 필요하면 먼저 현재 스키마를 확인하고 마이그레이션을 별도로 작성하세요.

먼저 할 일:
1. 대상 프로젝트에서 `git status`를 확인하세요.
2. 대상 프로젝트의 프론트엔드 라우팅, 관리자 페이지, DB 스키마, API 구조를 좁게 확인하세요.
3. 이식 범위를 아래 순서로 나누세요.

이식 우선순위:
1. 메뉴 구조
   - 관리자에서 1단/2단/3단 메뉴를 추가, 수정, 숨김, 정렬할 수 있게 합니다.
   - 메뉴 항목은 일반 링크, 동적 페이지, 게시판, 이미지 페이지, 갤러리, 유튜브, 외부 링크 등을 지원하도록 설계합니다.
   - 참고 파일: `MenuEditPanel.tsx`, `menu-edit/*`, `server/routers/cms/menus.ts`

2. 서브페이지 레이아웃
   - 상단 헤더, breadcrumb, 좌측 패널, 본문 영역 구조를 유지합니다.
   - 2단 메뉴를 눌렀을 때 자식 메뉴가 있으면 적절한 첫 하위 페이지로 이동하거나, 내용이 있으면 해당 내용을 보여주는 흐름을 정리합니다.
   - 참고 파일: `SubPageLayout.tsx`, `DynamicPage.tsx`, `SiteHeader.tsx`

3. 동적 페이지 콘텐츠
   - 관리자에서 등록한 콘텐츠가 실제 페이지에 표시되도록 합니다.
   - 이미지형, 에디터형, 게시판형, 자유게시판형, 갤러리형, 유튜브형 콘텐츠를 구분합니다.
   - 참고 파일: `dynamic-page/*`, `server/routers/cms/content.ts`, `server/db/content.ts`

4. 게시판형 UI
   - 공지사항, 주보 보기, 자막 신청, 탐방 신청, 주보 광고신청은 목록형 게시판 UI를 우선 적용합니다.
   - 제목을 누르면 상세/미리보기/첨부가 열리도록 합니다.
   - 로그인 필요 여부는 기능별로 다르게 둡니다.
   - 주보 광고신청과 자막 신청은 일반 성도가 작성할 수 있고, 탐방 신청은 비로그인 사용자도 작성할 수 있게 설계합니다.
   - 참고 파일: `BoardContent.tsx`, `FreeBoardContent.tsx`, `AdminSupportRequestsTab.tsx`

5. 최근행사사진/갤러리
   - 목록은 사진 썸네일 그리드 형태로 보여줍니다.
   - 게시물 상세는 이미지가 세로로 자연스럽게 나열되고, 클릭 시 크게 볼 수 있게 합니다.
   - 모바일에서는 이미지가 화면 폭에 맞게 줄어들어야 합니다.
   - 참고 파일: `GalleryContent.tsx`, `GalleryEditPanel.tsx`, `Lightbox.tsx`

6. 관리자 CMS
   - 관리자 페이지에서 메뉴, 콘텐츠, 주보, 팝업, 성도, 접수 관리 등을 한 화면에서 관리할 수 있게 합니다.
   - 대량 데이터가 생길 수 있는 영역은 검색, 상태 필터, 카테고리 필터, 접기/펼치기 또는 페이지네이션을 고려합니다.
   - 참고 파일: `Admin.tsx`, `Admin*Tab.tsx`

7. 서버/API/DB
   - 대상 프로젝트의 서버 구조에 맞게 라우터와 DB 접근 함수를 이식합니다.
   - 기쁨의교회 코드는 tRPC, Drizzle 기반입니다. 대상 프로젝트가 다르면 API 계약만 참고하고 구현은 대상 구조에 맞춥니다.
   - 참고 파일: `server/routers/*`, `server/db/*`, `drizzle/schema.ts`

검증:
- 타입 체크를 실행하세요.
- 테스트가 있으면 관련 테스트를 실행하세요.
- 빌드를 실행하세요.
- 브라우저에서 관리자 메뉴 편집, 게시판 목록/상세, 이미지 업로드/표시, 모바일 화면을 확인하세요.
- 검증하지 못한 항목은 최종 보고에 명확히 적으세요.
