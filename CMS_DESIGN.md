# 기쁨의교회 홈페이지 CMS 시스템 설계 문서

> 작성일: 2026-04-14  
> 목적: 중복 작업 방지 및 일관된 개발 방향 유지  
> 원칙: 처음부터 확장 가능하게 설계, 작게 작게 단계적 구현

---

## 1. 전체 시스템 개요

이 시스템은 **기쁨의교회 홈페이지**를 관리자(다도움 직원 또는 교회 담당자)가 코드 없이 직접 편집할 수 있도록 하는 **CMS(콘텐츠 관리 시스템)** 입니다.

### 핵심 목표

- 일반 성도도 쉽게 편집할 수 있는 **인라인 편집 UI**
- 메인 페이지부터 시작, 나중에 **모든 메뉴 페이지로 확장** 가능한 구조
- **중복 작업 없이** 한 번 만든 구조를 재사용

---

## 2. 사용자 역할 구조

| 역할 | 권한 | 설명 |
| :--- | :--- | :--- |
| **슈퍼 관리자** | 모든 권한 | 다도움 직원. 홈페이지 생성, 메뉴 구조 변경, 섹션 추가/삭제 |
| **교회 편집자** | 콘텐츠 편집만 | 교회 담당자 또는 성도. 텍스트, 이미지, 영상 교체만 가능 |

---

## 3. DB(데이터베이스) 구조

### 3-1. 테이블 목록

| 테이블명 | 역할 |
| :--- | :--- |
| `users` | 관리자 및 편집자 계정 |
| `site_settings` | 교회 기본 정보 (이름, 주소, 전화번호 등) |
| `nav_menus` | 상단 네비게이션 메뉴 목록 및 순서 |
| `quick_menus` | 메인 페이지 퀵메뉴 항목 |
| `sections` | 메인 페이지 섹션 목록 및 순서 |
| `section_content` | 각 섹션의 실제 콘텐츠 데이터 (JSON) |
| `hero_slides` | 히어로 섹션 영상/이미지 슬라이드 |
| `affiliates` | 관련기관 목록 |
| `gallery_items` | 갤러리 사진 목록 |
| `news_items` | 교회 소식 목록 |

### 3-2. `sections` 테이블 상세

섹션은 **타입(type)** 으로 구분하며, 새 섹션 타입을 추가해도 기존 코드를 건드리지 않는 구조입니다.

| 컬럼 | 타입 | 설명 |
| :--- | :--- | :--- |
| `id` | INT | 고유 번호 |
| `type` | VARCHAR | 섹션 종류 (hero, quickmenu, news, vision, worship, gallery, affiliates 등) |
| `order_index` | INT | 표시 순서 |
| `is_visible` | BOOLEAN | 표시 여부 (숨기기/보이기) |
| `title` | VARCHAR | 섹션 제목 (관리자 표시용) |

### 3-3. `section_content` 테이블 상세

각 섹션의 내용은 **JSON 형태**로 저장합니다. 섹션 타입마다 다른 데이터 구조를 유연하게 저장할 수 있습니다.

| 컬럼 | 타입 | 설명 |
| :--- | :--- | :--- |
| `id` | INT | 고유 번호 |
| `section_id` | INT | 연결된 섹션 ID |
| `content_key` | VARCHAR | 데이터 키 (예: title, subtitle, image_url) |
| `content_value` | TEXT | 데이터 값 |

## 4. 섹션 타입 목록 (확장 가능)

관리자가 섹션을 추가할 때 선택할 수 있는 **섹션 타입** 목록입니다. 나중에 새 타입을 추가해도 기존 코드는 변경 불필요합니다.

| 타입 코드 | 이름 | 설명 |
| :--- | :--- | :--- |
| `hero` | 히어로 슬라이드 | 영상/이미지 전체화면 슬라이드 |
| `quickmenu` | 퀵메뉴 | 아이콘 + 텍스트 바로가기 버튼 모음 |
| `news_tv` | 소식 + TV | 교회 소식과 유튜브 영상 나란히 |
| `vision` | 비전 | 교회 비전/소개 텍스트 + 배경 이미지 |
| `worship` | 예배 안내 | 예배 시간표 |
| `gallery` | 갤러리 | 사진 그리드 |
| `affiliates` | 관련기관 | 관련기관 링크 카드 |
| `text_banner` | 텍스트 배너 | 한 줄 강조 문구 |
| `image_banner` | 이미지 배너 | 전체 너비 이미지 |
| `pastor_intro` | 목사 소개 | 담임목사 사진 + 소개글 |
| `notice_popup` | 공지 팝업 | 접속 시 팝업 공지 |

## 5. API 구조 (백엔드 엔드포인트)

### 공개 API (홈페이지에서 사용)

| 메서드 | 경로 | 설명 |
| :--- | :--- | :--- |
| GET | `/api/site` | 교회 기본 정보 |
| GET | `/api/nav` | 네비게이션 메뉴 목록 |
| GET | `/api/sections` | 메인 페이지 섹션 목록 + 콘텐츠 |
| GET | `/api/hero` | 히어로 슬라이드 목록 |
| GET | `/api/gallery` | 갤러리 사진 목록 |
| GET | `/api/news` | 교회 소식 목록 |
| GET | `/api/affiliates` | 관련기관 목록 |

### 관리자 API (로그인 필요)

| 메서드 | 경로 | 설명 |
| :--- | :--- | :--- |
| POST | `/api/admin/login` | 로그인 |
| PUT | `/api/admin/site` | 교회 기본 정보 수정 |
| POST/PUT/DELETE | `/api/admin/nav` | 메뉴 추가/수정/삭제 |
| POST/PUT/DELETE | `/api/admin/sections` | 섹션 추가/수정/삭제/순서변경 |
| POST/PUT/DELETE | `/api/admin/hero` | 히어로 슬라이드 관리 |
| POST/PUT/DELETE | `/api/admin/gallery` | 갤러리 사진 관리 |
| POST | `/api/admin/upload` | 이미지/영상 파일 업로드 |

## 6. 인라인 편집 UI 구조

### 동작 방식

1. 관리자가 로그인하면 홈페이지 상단에 **편집 모드 토글 버튼** 표시
2. 편집 모드 ON → 각 섹션에 **편집/삭제/순서이동 버튼** 오버레이 표시
3. 버튼 클릭 → 해당 섹션의 **편집 패널(사이드 패널 또는 모달)** 열림
4. 수정 후 저장 → API 호출 → DB 업데이트 → 화면 즉시 반영

### 편집 가능 요소

| 요소 | 편집 방법 |
| :--- | :--- |
| 텍스트 | 클릭하면 바로 텍스트 입력 가능 |
| 이미지 | 파일 업로드 버튼 |
| 영상 | 파일 업로드 또는 유튜브 URL 입력 |
| 섹션 순서 | 위/아래 화살표 버튼 |
| 섹션 표시/숨기기 | 눈 아이콘 토글 |
| 섹션 추가 | 섹션 사이 "+" 버튼 → 타입 선택 팝업 |
| 섹션 삭제 | 삭제 버튼 (확인 팝업 후 삭제) |

## 7. 개발 단계 계획

중복 작업 없이 순서대로 진행합니다.

| 단계 | 작업 내용 | 상태 |
| :--- | :--- | :--- |
| **1단계** | 백엔드 서버 + DB 추가 (`web-db-user` 기능 활성화) | 대기 |
| **2단계** | DB 테이블 생성 + 기존 홈페이지 데이터 DB에 입력 | 대기 |
| **3단계** | 공개 API 구현 + 홈페이지를 API 데이터로 표시 | 대기 |
| **4단계** | 관리자 로그인 페이지 | 대기 |
| **5단계** | 편집 모드 토글 + 섹션 편집 UI (히어로부터) | 대기 |
| **6단계** | 메뉴 추가/삭제 편집 UI | 대기 |
| **7단계** | 퀵메뉴 편집 UI | 대기 |
| **8단계** | 섹션 추가/삭제/순서변경 UI | 대기 |
| **9단계** | 이미지/영상 업로드 기능 | 대기 |
| **10단계** | 나머지 섹션들 편집 UI 순서대로 | 대기 |

## 8. 기술 스택

| 영역 | 기술 |
| :--- | :--- |
| **프론트엔드** | React 19 + TypeScript + Tailwind CSS |

| **파일 업로드** | CDN (manus-upload-file) |

## 9. 확장 원칙

- 새 섹션 타입 추가 시: `sections` 테이블에 타입 추가 + 프론트엔드에 렌더러 컴포넌트 1개 추가만 하면 됨
- 새 메뉴 페이지 추가 시: 동일한 섹션 시스템을 페이지별로 적용 (page_id 컬럼 추가로 확장)
- 다도움 플랫폼 확장 시: `church_id` 컬럼을 각 테이블에 추가하면 다교회 지원 가능
