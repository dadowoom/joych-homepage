# 홈페이지 안정화 점검 기록

작성일: 2026-05-21  
대상 브랜치: `codex/social-login`  
목적: 기존 UI를 유지하면서 기능 안정성, 보안 의존성, 최종 도메인 전환 준비, 관리자 기능 위험 지점을 점검한다.

## 이번 작업에서 직접 보강한 항목

| 구분 | 내용 | 처리 |
| --- | --- | --- |
| 시설 예약 | 반려된 예약 시간이 사용자 신청 화면에서 계속 막혀 보일 수 있음 | `rejected` 예약은 `cancelled`와 동일하게 예약 가능 시간 계산에서 제외 |
| 시설 운영시간 | 관리자에서 같은 시설/요일 운영시간 저장 시 중복 행이 생길 수 있음 | 저장 시 해당 시설/요일 기존 행 삭제 후 재삽입, DB 유니크 제약 추가 |
| 시설 운영시간 | 이미 중복 데이터가 있는 운영 DB에서 유니크 제약 추가가 실패할 수 있음 | 마이그레이션에서 최신 행만 남기고 중복 정리 후 유니크 제약 추가 |
| 시설 사진 | 신규 시설 등록 시 사진 미리보기만 되고 실제 업로드가 누락될 수 있음 | 신규 시설 생성 후 임시 보관한 사진 데이터를 서버에 업로드하도록 연결 |
| 보안 의존성 | `pnpm audit` 기준 27개 취약점 경고 | Vite/Tailwind/PostCSS/Vitest/pnpm 업데이트 및 안전 override로 0개까지 정리 |

## 최종 도메인 전환 체크리스트

최종 도메인이 확정되면 아래 항목을 한 번에 바꿔야 한다. 현재 운영/심사 도메인은 `https://dadowoomtest.co.kr` 기준이다.

| 항목 | 현재 위치 | 전환 시 확인 |
| --- | --- | --- |
| 공개 기준 URL | 운영 `.env`의 `PUBLIC_URL_BASE` | 최종 도메인으로 변경 |
| 구글 간편가입 | Google Cloud OAuth 승인된 원본/리디렉션 URI | `/api/member-oauth/google/callback` 포함 최종 도메인 등록 |
| 카카오 간편가입 | Kakao Developers 카카오 로그인 리디렉션 URI | `/api/member-oauth/kakao/callback` 포함 최종 도메인 등록 |
| 카카오 지도 | Kakao Developers 플랫폼 Web 사이트 도메인 | 최종 도메인 등록 |
| 사이트맵 | `client/public/sitemap.xml` | 모든 `dadowoomtest.co.kr` URL을 최종 도메인으로 변경 |
| robots | `client/public/robots.txt` | `Sitemap:` URL 최종 도메인으로 변경 |
| README/운영문서 | `README.md`, 운영 문서 | 표기 도메인 최신화 |

도메인이 확정되기 전에는 위 값을 임의 변경하지 않는다. OAuth와 카카오 심사가 깨질 수 있기 때문이다.

## 관리자 기능 정적 점검 결과

| 영역 | 확인 내용 | 결과 |
| --- | --- | --- |
| 관리자 진입 | `/admin`은 공개 관리자 경로로 쓰지 않고, 실제 경로는 `/admin_joych_2026` | 정상 |
| 관리자 API 권한 | 메뉴, 콘텐츠, 업로드, 공지, 시설, 예약, 선교보고 관리 라우터가 `adminProcedure` 사용 | 정상 |
| 성도 관리 | 삭제 버튼은 실제 삭제가 아니라 `withdrawn` 상태 처리 | 정상 |
| 예약 관리 | 승인/반려 API는 관리자 전용, 사용자 예약 생성은 승인 성도 전용 | 정상 |
| 시설 관리 | 신규 시설 사진 업로드 누락 가능성 발견 | 수정 완료 |
| 메뉴 관리 | 생성/수정/삭제/정렬 후 공개 메뉴 invalidate 처리 있음 | 정상 |
| 페이지 콘텐츠 관리 | 잘못된 JSON은 서버 검증에서 차단 | 정상 |
| 업로드 | 관리자 전용, MIME/파일 시그니처/크기 제한 확인 | 정상 |

## 남겨둔 운영 메모

- 카카오 비즈니스 심사 대기 중에는 푸터 사업자/운영주체 문구를 변경하지 않는다.
- 최종 도메인 확정 전까지 OAuth/지도 도메인 설정은 `dadowoomtest.co.kr` 기준으로 유지한다.
- 패키지 업데이트 후 `@builder.io/vite-plugin-jsx-loc`의 Vite peer warning은 남을 수 있다. 빌드/테스트 통과 여부로 실제 영향 확인이 필요하다.
