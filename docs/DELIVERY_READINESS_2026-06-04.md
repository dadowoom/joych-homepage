작성일: 2026-06-04  
대상 브랜치: `codex/signup-hardening`  
목적: 납품 전 남은 리스크를 한눈에 보고, 실제 검수/배포/인수인계 기준으로 마감한다.

## 현재 결론

현재 사이트는 기본 기능, 타입 체크, 테스트, 프로덕션 빌드가 통과하는 상태다.  
다만 납품 수준으로 보려면 자동 검증만으로는 부족하고, 공개 화면 버튼/폼, 관리자 저장 후 공개 반영, 회원 승인 상태별 접근, 운영 데이터 정리가 끝까지 확인되어야 한다.

## 수동 검수 체크리스트

### 공개 사이트

- [ ] 홈 첫 화면 히어로 영상이 의도한 최신 영상만 노출된다.
- [ ] 새로고침 후 예전 메뉴/예전 영상이 튀어나오지 않는다.
- [ ] 상단 메뉴와 하위 메뉴가 PC/모바일에서 정상 열린다.
- [ ] 홈 버튼, 새가족 등록, 예배 안내 버튼이 의도한 곳으로 이동한다.
- [ ] 교회소개, 조이풀TV, 커뮤니티, 행정지원 하위 메뉴가 모두 응답한다.
- [ ] 오시는 길 지도와 길찾기가 정상 동작한다.
- [ ] 푸터 정보가 모바일에서도 잘리지 않는다.

### 회원/권한

- [ ] 일반 회원가입 후 승인 대기 상태가 된다.
- [ ] Google 가입 후 승인 대기 상태가 된다.
- [ ] Kakao 가입 후 승인 대기 상태가 된다.
- [ ] 승인 전에는 성도 전용 기능 접근이 제한된다.
- [ ] 관리자 승인 후 로그인과 성도 전용 기능 접근이 가능하다.
- [ ] 반려/탈퇴 상태에서는 로그인 또는 전용 기능 접근이 제한된다.
- [ ] 생선 간증 글쓰기/댓글은 승인된 성도만 가능하다.

### 관리자

- [ ] 관리자 로그인 제한 정책이 과하게 막히지 않는다.
- [ ] 관리자 대시보드 탭 이동과 `?tab=` 주소가 정상 유지된다.
- [ ] 메뉴 수정 후 공개 메뉴에 반영된다.
- [ ] 히어로 슬라이드 수정 후 공개 홈에 반영된다.
- [ ] 공지/갤러리/시설/예약/성도 관리 저장이 정상 동작한다.
- [ ] 소셜 가입자가 성도 관리 목록에 표시되고 승인/반려 처리된다.

## 배포 전 확인 순서

1. `git status --short`로 작업트리 확인
2. `pnpm check`
3. `pnpm test`
4. `pnpm build`
5. 운영 서버 백업
6. 서버 pull/build/restart
7. 공개 사이트와 관리자 주소 HTTP 200 확인
8. PM2 로그 확인
9. 핵심 수동 검수

## 남은 리스크

- 자동 테스트는 통과했지만, 실제 화면에서 모든 버튼/폼을 누르는 수동 검수는 아직 필요하다.
- 운영 DB에 남아 있는 히어로/메뉴 과거 데이터는 코드 검증만으로 판단할 수 없다.
- 최종 도메인 전환 시 Google/Kakao OAuth, Kakao Map, sitemap, robots, `PUBLIC_URL_BASE`를 함께 바꿔야 한다.
- 운영 비밀번호와 토큰 교체는 최종 전환 직전에 반드시 별도 작업으로 처리해야 한다.
