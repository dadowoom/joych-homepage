작성일: 2026-06-03  
대상 브랜치: `codex/signup-hardening`

## 검수 기준

- 사용자가 누르는 버튼은 실제 이동, 실제 저장, 실제 외부 앱 실행 중 하나로 이어져야 한다.
- 아직 온라인 처리하지 않는 기능은 "완료"처럼 보이면 안 된다.
- 기존 디자인은 유지하고, 기능 동작과 안내 문구만 점검한다.

## 수동 브라우저 검수 필요

자동 테스트와 빌드가 통과해도 아래는 사람이 실제 화면에서 확인해야 한다.

1. 홈 첫 화면에서 히어로 영상이 최신 영상만 보이는지
2. 홈 버튼 2개가 의도한 페이지로 이동하는지
3. 상단 메뉴와 하위 메뉴가 새로고침 후에도 같은 구조로 보이는지
4. 관리자에서 메뉴/페이지를 수정한 뒤 공개 화면에 반영되는지
5. 승인 대기 성도가 로그인/글쓰기/시설예약을 못 하는지
6. 승인된 성도가 생선 간증 글쓰기와 댓글을 할 수 있는지
7. 모바일에서 푸터, 오시는 길, 회원가입, 관리자 목록이 가로로 밀리지 않는지

## 다음 구현 후보

우선순위는 아래 순서가 적절하다.

1. 히어로 영상 운영 DB 정리
2. 상단 메뉴와 CMS 메뉴 충돌 정리
3. 자막 신청/탐방 신청을 실제 접수 기능으로 확장
4. 기쁨톡을 만들지, 생선 간증 중심으로 유지할지 결정
5. 주보 PDF 업로드/다운로드 관리 추가
6. 일본어 번역 토글 및 AI 설교 번역은 별도 대형 작업으로 분리

## 결론

지금 당장 큰 구조를 바꾸기보다, 공개 화면에서 "완료처럼 보이지만 실제 저장되지 않는 기능"을 먼저 줄이는 것이 납품 리스크를 가장 빠르게 낮춘다.
