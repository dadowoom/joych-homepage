## 다음 작업 우선순위

### 1. 공개 화면 버튼/폼 전수 검수

가장 먼저 해야 한다. 자동 테스트로는 사용자가 실제로 누르는 버튼의 체감 문제를 모두 잡을 수 없다.

우선 확인할 화면:

- 홈 히어로 버튼: 새가족 등록, 예배 안내
- 교회소개 하위 메뉴 전체
- 조이풀TV 목록, 상세, 옛 VOD 링크
- 커뮤니티: 기도 요청, 생선 간증, 기쁨톡, 교회 소식
- 행정지원: 새가족 안내, 시설 예약, 자막 신청, 온라인 사무국, 탐방 신청, 기부금 영수증
- 오시는 길: 지도 표시, 출발지 검색, 현재 위치 길찾기, 카카오맵 바로가기

특히 `client/src/pages/CommunityExtra.tsx`의 자막 신청, 온라인 사무국, 탐방 신청, 기부금 영수증은 아직 실제 온라인 접수 저장이 아니라 행정실 문의 안내 중심이다. 납품 기준에서는 둘 중 하나로 결정해야 한다.

- 실제 접수 기능으로 만들기
- 온라인 접수 메뉴에서 제외하거나, 명확히 전화 문의 페이지로 유지하기

### 2. 히어로 영상 운영 데이터 정리

사용자가 "예전 영상이 다시 나온다"고 느꼈던 영역이다. 프론트 빌드보다 운영 DB의 히어로 슬라이드 목록과 공개 여부가 더 중요하다.

확인할 것:

- 관리자 히어로 슬라이드 목록
- `isActive`, `sortOrder`, 영상 URL, 포스터 URL
- 운영 DB에 예전 영상이 활성 상태로 남아 있는지
- 새로고침 후 같은 순서로 보이는지

완료 기준:

- 홈 첫 화면에 의도한 최신 영상만 노출
- 새로고침 후 과거 영상 재노출 없음
- 관리자에서 비활성 처리한 슬라이드는 공개 화면에 나오지 않음

### 3. CMS 기준 데이터 통합

아직 코드에 박힌 안내성 페이지와 관리자에서 만든 페이지가 섞여 있다. 납품 후 교회 담당자가 직접 운영하려면 CMS 기준을 더 분명히 해야 한다.

우선순위:

- 상단 메뉴와 하위 메뉴 기준 데이터 정리
- 관리자 생성 페이지 주소, 공개 링크, 새로고침 라우팅 확인
- 코드에 고정된 페이지 중 자주 바뀔 내용만 CMS로 이전
- 섬기는 사람들, 부교역자, 교회학교, 부서 소개 데이터의 관리자 관리 범위 확정

주의:

- 모든 페이지를 무조건 CMS로 옮기면 오히려 품질이 떨어질 수 있다.
- 자주 바뀌는 콘텐츠는 CMS, 고정된 구조 페이지는 코드 유지가 더 안정적이다.

### 4. 회원가입/승인/권한 회귀 검수

현재 코드상으로는 승인 전 로그인이 막히고, 승인된 성도만 성도 전용 기능을 쓰도록 되어 있다. 그래도 운영 전에는 실제 계정으로 다시 봐야 한다.

검수 시나리오:

- 일반 회원가입 후 승인 대기 상태 확인
- Google 가입 후 승인 대기 상태 확인
- Kakao 가입 후 승인 대기 상태 확인
- 승인 전 로그인 차단 확인
- 관리자 승인 후 로그인 가능 확인
- 반려/탈퇴 상태에서 로그인 차단 확인
- 승인된 성도만 생선 간증 글쓰기/댓글 가능 확인
- 관리자 성도 목록에 일반/Google/Kakao 가입자가 모두 보이는지 확인

### 5. 행정지원 접수 기능 확장

현재 완료된 접수:

- 기도 요청
- 새가족 등록

추가 후보:

- 자막 신청
- 탐방 신청
- 상담 신청
- 봉사 신청
- 증명서/기부금 영수증 신청

주의:

- 기부금 영수증은 주민등록번호 등 민감 정보 이슈가 크다.
- 온라인으로 받으려면 저장 범위, 마스킹, 관리자 접근 권한, 삭제 정책을 먼저 정해야 한다.
- 지금은 전화 문의 유지가 더 안전할 수 있다.

### 6. 일본어 번역/AI 설교 번역은 별도 5.5 작업으로 분리

이 기능은 충분히 가능하지만, 현재 납품 안정화와 섞으면 위험하다.

권장 방향:

- 홈페이지 기본 문구: CMS/정적 문구 기반 한국어-일본어 번역 테이블
- 공지/소개/설교 설명: 관리자에서 AI 번역 생성 후 사람이 확인하고 공개
- 유튜브 설교: 영상 링크 등록 후 음성 추출, 전사, 일본어 번역, 요약 제공
- API 후보: OpenAI 최신 음성 전사 + 번역 모델 조합

완료 기준은 "자동 번역이 바로 공개"가 아니라 "AI 초안 생성 후 관리자가 확인하고 공개"가 되어야 한다.
